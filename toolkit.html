Common questions

What tech stack does this assume?
Built on Bun (TypeScript) + SQLite + Ollama for local LLMs + Claude for cloud. The patterns and templates work with any LLM provider — the architecture is what matters, not the specific APIs.
Do I need a Mac?
We run on Mac Mini M4 Pro, but the system works on any machine that can run Ollama (Mac, Linux, Windows with WSL). The local LLM routing is optional — you can start cloud-only and add local later.
Is this a framework or a tutorial?
Neither. It's the actual config files, templates, and architecture docs from a production system. You'll adapt them to your use case, not install a package. Think "starter kit" not "framework."
How is this different from CrewAI / AutoGen / LangGraph?
Those are frameworks. This is the architecture that replaced them. We tried CrewAI — too much abstraction, not enough control over costs. This toolkit gives you the patterns for building your own lightweight system without framework lock-in.
What if I only want 2-3 agents?
Start there. The playbook explicitly recommends 2-3 agents first. The soul templates, budget routing, and tool limits all work at any scale. You don't need 16 agents to benefit from the architecture.
